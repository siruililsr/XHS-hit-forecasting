# 小红书爆款预测器 — 客户操作 & 评价指南

> 本系统为天津师范大学周边美食探店帖子提供发布前 / 发布后热度预测。以下请按步骤操作，每步附评价点供参考。

---

## 一、环境准备（5 分钟）

### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 启动应用
在项目目录下执行：
```bash
streamlit run app.py
```

浏览器会自动打开 `http://localhost:8501`。

> **评价点：** 安装是否顺畅？启动速度如何？依赖有没有版本冲突？

---

## 二、发前预测（核心功能）

### 操作
1. 在左侧「📝 发布前预测」Tab 中输入：
   - **标题**：`天师大后街这家烤肉真的绝了!! 人均才30`
   - **正文**：任意 200~400 字探店描述，或使用我们提供的示例文案
   - **标签**：`天津师范,校园美食,穷鬼套餐,烤肉推荐,大学城美食`
2. 点击「预测（发前）」按钮。

### 预期结果
- 页面展示三类概率分布（低热度 / 中热度 / 高热度）三色进度条
- 给出预测结论（大字标签，一目了然）
- 下方展示改进建议（如正文长度、标题情感、标签数量等）

### 请评价
| 评价维度 | 说明 |
|----------|------|
| 响应速度 | 点击按钮后多久出结果？ |
| 结果清晰度 | 三段式布局（概率条 → 大字标签 → 改进建议）是否直观？ |
| 改进建议实用性 | 给的建议是否具体、可操作？还是泛泛而谈？ |
| 准确度预期 | 是否清楚知道这个预测仅作参考、不是定论？ |

## 三、封面多模态预测（图文联合）

### 操作
1. 在页面上传一张美食封面图（项目提供了演示用图 `演示封面_烤肉.jpg`）
2. 系统自动启动多模态融合：CNN 分析图片 + CLIP 检测图文匹配
3. 观察结果区域新增的多模态信息

### 预期结果
- 多模态融合卡片：综合 CNN 和文本的判断
- CLIP 图文相似度分数及匹配结论
- CNN 和 GBM 各自的预测对比
- 如果图文不一致，会弹出警告

### 请评价
| 评价维度 | 说明 |
|----------|------|
| 封面上传体验 | 支持哪些格式？上传后多久显示预览？ |
| 多模态结果可读性 | CNN/Text/CLIP 三项分开展示，是否容易理解？ |
| 图文不符警告 | 这个功能的实际价值如何？是否可能避免用户发布图文不符的内容？ |
| 综合判断说服力 | 融合结果（0.4×CNN + 0.6×Text）的加权逻辑是否合理？ |

## 四、发后重测（有评论后复测）

### 操作
1. 切换到「🔄 发后重测」Tab
2. 观察标题、正文、标签是否自动从发前 Tab 带入（**同步测试**）
3. 手动修改正文中的一个字 → 观察是否被覆盖（**dirty flag 测试**）
4. 填入评论数据：
   ```
   求地址!!
   已收藏
   好吃!!!
   人均多少呀
   周末去!
   ```
5. 点击「重测（发后）」按钮

### 预期结果
- F1 从 0.408 提升到 0.621
- 评论数据生效后概率分布发生变化
- Tab 同步正常：自动带入 → 手动修改不被覆盖

### 请评价
| 评价维度 | 说明 |
|----------|------|
| Tab 同步体验 | 发前输入的内容是否顺利带到发后？手动修改是否被保护？ |
| F1 差异感知 | 是否明显感受到"有评论数据后模型更准"？ |
| 评论输入便捷性 | 评论框的设计是否好用？ |

## 附录：已知局限（请向客户如实说明）

1. **发前 F1=0.408**：在没有评论数据的情况下，模型预测能力有限，仅作参考。
2. **训练数据 649 条**：覆盖的场景有限，全为天津师大周边美食探店。
3. **高热度预测偏保守**：因训练数据中爆款帖子本身占少数，模型倾向于给出中低热度判断。
4. **封面图需自行准备**：目前不包含图片爬取或自动生成封面功能。
5. **仅支持中文**：标题和正文须为中文，标签用逗号或空格分隔。
